# Planfit 프로젝트 README

> 운동 관리 웹앱 Planfit UI/UX 구현 및 정보 구조 요약 문서

---

## 1. 프로젝트 개요

* 프로젝트명: Planfit
* 목적: 운동 추천, 실행, 기록 관리를 위한 웹 서비스 구현
* 환경: 모바일 중심 UI 설계
* 특징: 페이지별 CSS/JS 분리 및 직관적인 화면 구조

---

## 4. 전체 UI 구조

### Header

* 로고 / 검색 / 설정 / 메뉴 / 사용자 아이콘
* 모든 페이지 공통 적용

### Bottom Navigation

* 홈 / 메뉴 / 시작 / 마이
* 주요 화면 빠른 이동 기능 제공

### Footer

* 이용약관 / 개인정보처리방침 / 안내 문구

---

## 5. 사용자 흐름 (User Flow)

```
홈 → 메뉴 → 루틴 → 운동 시작 → 기록 → 마이 → 설정
         ↘ 로그인 / 회원가입
```

사용자의 운동 이용 흐름을 기준으로 화면 구조를 설계하였다.

---

## 6. 화면별 주요 기능

### Home (01_index.html)

* 오늘의 추천 루틴 제공
* 운동 시작 버튼
* 가이드 및 등록 카드

### Menu (02_menu.html)

* 운동/루틴 탐색
* 카테고리 분류

### Routine (03_routine.html)

* 루틴 상세 정보 제공
* 운동 구성 확인

### Start (04_start.html)

* 운동 진행 화면
* 세트 및 휴식 타이머

### Record (05_record.html)

* 운동 기록 저장 및 확인
* 통계 요약

### MyPage (06_mypage.html)

* 사용자 상태 표시
* 목표 및 정보 관리

### Settings (07_settings.html)

* 알림 및 환경 설정

### Login / Signup

* 사용자 인증 및 계정 관리

---

## 7. 구현 규칙 (요약)

* 네이밍: 번호 + 의미 기반 파일명 사용
* HTML: 시맨틱 태그, 2칸 들여쓰기
* CSS: 공통(style.css) + 페이지별 분리
* JS: 기능별 파일 분리
* 접근성: aria-label, alt 속성 적용

## 10. 구조 설계 원칙

1. 사용자 흐름 중심 설계

* 홈 → 탐색 → 실행 → 기록 → 관리 흐름을 기준으로 IA를 구성하였다.

2. 역할 기반 자원 분리

* HTML, CSS, JS, 이미지 리소스를 분리하여 유지보수성을 확보하였다.

3. 페이지 번호 체계 적용

* 01~09 번호 체계를 적용하여 화면 순서를 명확히 하였다.

4. 전역 UI 일관성 유지

* Header / Bottom Navigation / Footer를 공통 구조로 유지하였다.
